## 📥 Формат входных данных

Все алгоритмы используют **единый формат** ввода через файл `input.txt`:

### Для сортировки:
N # количество элементов (1 ≤ N ≤ 10⁵)
массив_1 # первый массив (индекс 1)
массив_2 # второй массив (индекс 2)
массив_3 # третий массив (индекс 3)
...

text

**Пример `input.txt`:**
12
5 2 4 6 11 9 78 56 3 4 9 15 59 33
8 6 5 9 12 173 4 9 15 59 33 89 45 12
1 2 3 -3 4 4 -5 6 56 78 9 15 59 33

text

### Для поиска (бинарный, тернарный, экспоненциальный):
N # количество элементов
массив # отсортированный массив
target # искомое число

text

**Пример:**
5
1 3 5 7 9
7

text

### Для Two Pointers:
N # количество элементов
массив # массив с нулями или элементами для удаления
element # элемент для удаления (только для remove_element)

text

💻 Особенности реализаций
1. Сортировка слиянием (оптимизированная)
Реализованные оптимизации:

Ранняя сортировка подмассивов из 2 элементов (без рекурсии)

Быстрый путь: копирование без сравнений для уже отсортированных частей

Использование встроенной copy() вместо ручного копирования

Минимизация выделений памяти (один result на уровень)

Теоретическое преимущество:

На случайных данных: ожидается выигрыш ~20-30% за счёт copy() и меньшего числа рекурсивных вызовов

На отсортированных данных: значительное ускорение благодаря быстрому пути

На обратно отсортированных: умеренный выигрыш (~15-25%)

2. Быстрая сортировка
В проекте представлены две реализации:

Моя реализация (quick_sort_mine.go)
Особенности:

Использует схему разбиения Хоара

Выбор опорного элемента через медиану из трёх

Собственная логика поиска элементов с использованием флагов (flagLeft, flagRight)

Отличие от стандартной реализации:

Вместо классических указателей i и j используется более сложная логика с флагами

Поиск элементов для обмена выполняется с отслеживанием состояния через булевы флаги

Оценка эффективности:

Код оказался менее эффективным, чем стандартная реализация

Дополнительные проверки флагов создают накладные расходы

Больше операций на каждой итерации

Почему эта реализация интересна:

Оригинальная реализация.

go
// Ключевая особенность: флаги для поиска элементов
for left <= right {
    if arr[left] < pivot {
        left++
    } else {
        if flagLeft {
            tempLeft = left
            flagLeft = false
        }
    }
    if arr[right] > pivot {
        right--
    } else {
        if flagRight {
            tempRight = right
            flagRight = false
        }
    }
    if flagLeft == false && flagRight == false {
        arr[tempRight], arr[tempLeft] = arr[tempLeft], arr[tempRight]
        flagLeft, flagRight = true, true
        left++
        right--
        tempLeft = left
        tempRight = right
    }
}
Стандартная реализация (quick_sort_standart.go)
Особенности:

Классическая схема разбиения Хоара

Выбор опорного элемента через медиану из трёх

Минимальное количество операций

Используется в реальных проектах

go
// Классическая реализация (лаконичная и эффективная)
func partition(arr []int, left, right int) int {
    pivot := arr[mediana(arr, left, right)]
    i := left - 1
    j := right + 1
    
    for {
        for i++; arr[i] < pivot; i++ {}
        for j--; arr[j] > pivot; j-- {}
        if i >= j {
            return j
        }
        arr[i], arr[j] = arr[j], arr[i]
    }
}
